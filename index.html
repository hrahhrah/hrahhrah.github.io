Team B

A software engineering for Data Science project

Project Purpose

The purpose of this project is to follow a typical software life cycle model through from beginning to end. We will focus on improving the current existing machine learning model to classify Amazon reviews using sentiment analysis, while using the OpenUp Agile methodology. We will also discuss wrapping the model in a user-friendly interface to allow quick and easy positive/negative predictions to new review text.

Goals

The goal of the project is to improve the accuracy and reliability of the existing machine learning model by following a typical software life cycle process. The current model is making predictions with an accuracy of ~60%, which is no better than random guessing when taking into account the spread of the data. The current model is exclusively predicting a rating of 1.0, which accounts for 60% of the entire dataset, and therefore results in the 60% accuracy.

We hope to improve both the accuracy and the AUC (area under the ROC curve). Since we will be transforming the target labels into binary (positive and negative) sentiment, accuracy alone will not prove the reliability of our model. Therefore we will also be looking at the ROC curve and specifically the AUC.


Requirements

  1. An algorithm will consume text review data and return a predictive score for the numerical rating of that review, either positive (1) or negative (0)
    1.1 The model must be a classification algorithm
  2. Training Model Accuracy will exceed 60% in order to improve on the existing model
  3. The algorithm will be accessed via API on a public website
  4. The User Interface for this API must have a text box where users can paste a review text and submit
  5. The API will return a score within a reasonable time frame after the review is submitted
  6. The User Interface will make the predicted binary score available to the user

Architecture

For this term project, our solution architecture is limited to MS Azure ML Studio. We will deliver our improved model and outputs via ML Studio directly.

In addition, we have drawn out what the architecture might look like for a production deployment. There, we would have users interact with a simple UI attached to API endpoint. In that UI, users can input review text and get a “pos/neg” sentiment returned. We would have our reference data in Azure storage and use ML Studio to build the model. The model and API would be deployed using an AKS (Azure Kubernetes Service), since we want to have a real-time interaction and response.

For more information about architecture and designs:
https://hrahhrah.github.io/Architecture_Design.pdf

Models

We are researching and experimenting with various two class algorithms and data processing steps such as feature hashing, text preprocessing, Latent Dirichlet Allocation, and extracting key phrases. A finalized model description with metrics and a diagram will be displayed here after testing and evaluation is completed.
